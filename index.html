Summary

We plan to implement and optimize a parallelized version of the PageRank algorithm using three complementary parallel programming models: OpenMP, MPI, and CUDA, to measure the importance of different nodes in a network. Our goal is to design and benchmark our parallel implementation and compare it against the sequential implementation and potentially other parallel implementations if time persists.

Background

PageRank, developed by Google founders, is a web search ranking algorithm based on numerical importance metrics. It treats the web as a graph where webpages are nodes and hyperlinks are edges. A webpage receives a high importance score if many pages link to it or if it is linked by highly ranked pages, factoring in both quantity and quality of links.

Pseudocode for the algorithm:

While not converged:
    For each node in graph:
        Compute new rank from in-neighbors
    Update ranks

While primarily used for webpage ranking, PageRank can also be applied to recommendation systems, citation analysis, and ranking papers or other networks.

The algorithm is well-suited for parallelism: each node's update can be computed independently per iteration, and the sparse matrix-vector multiplication and iterative power method are highly parallelizable.

Challenge

Parallelizing PageRank is challenging due to the memory-bound nature of the algorithm and the structure of real-world graphs. Each iteration involves scanning many edges, with limited computation per edge, making performance sensitive to memory bandwidth, memory access patterns, and communication overhead. We aim to study how different parallel paradigms handle these irregular, sparse, memory-bound workloads and what bottlenecks dominate in each model.

Workload

Within each iteration, node updates are independent if the previous iteration's vector is read-only, which allows parallelization across nodes. However, iterations are dependent on previous results, requiring synchronization between steps for correctness.

Constraints

PageRank is memory-bound rather than compute-bound. Large graphs require significant memory, and random access patterns can exceed cache capacity, causing memory bottlenecks. Performance is limited by memory bandwidth, cache efficiency, and the ability to fit the working set in fast-access memory.

Goals / Deliverables

Plan to Achieve

  • Sequential baseline implementation for correctness and speedup comparison
  • Three separate parallel implementations using OpenMP, MPI, and CUDA
  • Performance evaluation: scaling, GPU utilization, memory bandwidth, and communication overhead

Hope to Achieve

  • Hybrid parallelism (MPI + OpenMP or MPI + CUDA)
  • Asynchronous Gauss-Seidel PageRank algorithm to improve convergence and memory utilization

Platform Choice

We chose OpenMP for multicore CPU parallelism, MPI for distributed-memory scaling, and CUDA for GPU acceleration. Each platform fits naturally with PageRank's iterative power method and sparse matrix-vector multiplication operations, allowing us to explore scaling, performance bottlenecks, and memory-bound workload handling.
